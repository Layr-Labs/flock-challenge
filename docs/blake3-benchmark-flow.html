Flock BLAKE3 benchmark · implementation review map

Private input. Editable prover. Fixed verifier.

The benchmark times a submission-built Flock prover on fresh BLAKE3 compression inputs. A prebuilt verifier reconstructs the same witness, checks the proof, and writes the only score Yukon accepts.

Ranked invariant

The candidate may change the proving implementation, but it does not choose the scored input, measure its own time, load code into the trusted verifier, or write score.json.

The ranked protocol

Seven ordered moves from checkout to score.

GitHub Actions executes one exact submission commit. The trusted verifier owns every transition that affects the private challenge, wall-clock duration, proof verdict, or score.

What is measured

BLAKE3 compression proofs, not whole-message hashes.

Each operation proves one raw BLAKE3 compression with a generated chaining value, one 64-byte message block, and a counter. Input generation through proof-file write and worker exit is timed.

One generated operation

262,144 per timed proof
(cv: [u32; 8], message: [u32; 16], counter: u64, block_len: 64, flags: 11)

A private seed expands deterministically into the complete batch. Every timed trial uses a new seed.

Final proof check

A valid protocol proof must also bind the private witness.

The trusted commitment reconstruction is the benchmark-specific correctness check. It prevents a solver from proving a cheaper or unrelated statement.

01 / SUCCESSFUL EXIT

The timer stops only when the worker exits zero within the deadline. A ready message or partial file cannot stop it.

02 / BOUNDED FILE

Require a nonempty proof within the trusted byte limit before allocation and decoding.

03 / EXACT FORMAT

Require FLOCK magic, current version, R1CS flavor, fixed-width bincode, and no trailing bytes.

04 / REBUILT INPUT

Expand the same undisclosed seed into the exact 262,144 compression tuples.

05 / EQUAL COMMITMENT

Regenerate the pristine packed witness; require root and every PCS parameter to match.

06 / PRISTINE VERIFICATION

Run compiled-in Blake3Setup::verify under the fixed flock-bench-v0 transcript domain.

Score file

One typed JSON object, written after all checks.

The verifier discards 20 verified warm-up durations, linearly interpolates P10 over 100 measured durations, and serializes Rust ScoreFile and ScoreMetrics values with Serde.

Hardware and operational security

Official throughput belongs to one runner class.

Local Apple Silicon runs are useful for correctness and directional feedback. Absolute ranking comes from the dedicated M4 Pro because chip configuration, compiler output, thermals, and background load affect throughput.

Official GitHub runner

authoritative
  • Apple M4 Pro, 48 GB unified memory
  • 10 performance cores selected by default
  • arm64 macOS; validated run: 26.4 build 25E246
  • Rust 1.97.0; candidate target-cpu=native
  • Self-hosted label m4-pro; one benchmark at a time

Local Apple Silicon

directional
  • ./setup.sh checks arm64 macOS, Xcode Clang, Rust, checksum, and signing
  • ./benchmark.sh uses local performance cores
  • Compare changes only on the same quiet machine
  • The official M4 Pro result remains the ranked result
FAKE TIME OR SCORE

Worker output is ignored, the external verifier owns Instant, and Seatbelt prevents writes to root score.json.

CHEAPER STATEMENT

Trusted input and witness reconstruction makes a different seed, input, root, or PCS profile fail before protocol acceptance.

BACKGROUND WRITER

Seatbelt denies child processes; the timer waits for worker exit and no descendant may finish afterward.

STALE RESULT

benchmark.sh removes previous scores before execution. A failed run has no valid new score artifact.

VERIFIER REPLACEMENT

The checksum-pinned binary contains its own verifier imports and never loads candidate crate code.

HOST BOUNDARY

Seatbelt constrains the measured worker, not an entire VM. The dedicated runner must contain no unrelated secrets or workloads.
